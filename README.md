# Future-Library-Policy

https://taejeong0416.github.io/Future-Library-Policy/

초본(draft)과 최종본(final)의 변형 과정을 산출한 `report.json`을 **업로드해서 시각화**하는 정적(Static) 웹 뷰어입니다.
예시 로드를 누르면 리포지토리에 포함된 리포트를 예시로 볼 수 있습니다.

예시를 참고하여, 사용자는 다음 흐름으로 동일한 방식의 리포트를 만들 수 있습니다.

example_draft.txt  +  example_final.txt   (또는 본인 파일)\
            ↓\
compare_process.py 실행\
            ↓\
report.json 생성\
            ↓\
이 뷰어(index.html)에 업로드 → 결과 확인

# 예시 리포트 결과 요약
1) 표현 변화(겉모양)

문자 유사도 0.28 → 낮음

변화의 핵심은 삭제/추가가 아니라 치환(재작성)

전체 길이는 -103자(소폭 감소)

-> 같은 뼈대 위에서 문장을 갈아쓴 형태로 기록됨.

2) 의미 변화(키워드 기준)

TF-IDF 문서 유사도 0.18 → 낮음

초본 1블록 → 최종본 6블록으로 분할

초본과 강하게 겹치는 블록이 없음

-> 내용이 바뀌었다기보다 ‘표현과 구조가 재편성’되어 키워드 겹침이 낮아진 경우로 해석이 적절.

3) 구조 변화

제목 패턴이 없어 구조 지표는 사실상 비활성

“구조가 없다”가 아니라 자동 인식 가능한 표식이 없었다는 의미

4) 정보량 변화(프록시)

총 토큰 -32(약 6.6% 축소)

고유어 -16(어휘 다양성 소폭 감소)

숫자 동일(3→3)

따옴표 +4(증가), 괄호 -1(감소)

-> 괄호식 부연은 줄이고, 인용/강조 표현은 늘린 정리본에 가깝다.
